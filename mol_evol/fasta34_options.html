FASTA command line options

-a Show entire length of both sequences in alignment (fasta, ssearch)
-A Do full Smith-Waterman alignment for DNA (fasta)
-b # Number of high scores to display
-C # Length of sequence name labeling aligned sequences
-d # Number of alignments to display
-D Use DEBUG mode
-E # Expectation threshold for displaying scores, alignments
-f # Gap-open penalty (-10 default for proteins with default BLOSUM50 matrix)
-F # Expectation lower limit for displaying scores, alignments
-g # Gap-extension penalty (-2 default for proteins with default BLOSUM50 matrix)
-h # Frameshift between codon penalty (fastax, fasty, tfastx, tfasty)
-H Omit histogram (fasta programs); include histogram (pvcomp/mpcomp programs)
-i Search with reverse complement query only (fasta DNA, fastx, fasty)
-j # Frameshift within codon penalty (fasty, tfasty)
-J #:# First query, last query (pvcomp/mpcomp)
-K # Node buffer size (pvcomp/mpcomp)
-l file FASTLIBS library
-L Long sequence descriptions on alignments
-m # Alignment display options
-M #:# Library length range; only sequences within the residue range are considered
-n Query sequence is DNA
-N # Library segment length (examine long DNA sequences in overlapping lengths of # nt)
-o Turn-off band optimization (fasta)
-O file Output file name (most useful with MacOS9)
-p Query sequence is protein
-q/-Q Quiet - do not prompt for any input
-r #/# DNA match/mismatch scores (+5/-4 default)
-R file Score results file (for debugging)
-s file Scoring matrix: file name or BL50, BL62, BL80, P10, P20,P40,P120, P250
-S Treat lower case as low complexity
-t # NCBI genetic code translation table
-T # Number of workers (pvcomp/mpcomp, threaded)
-v Shuffle window size
-w Alignment display width (default 60)
-W Alignment context (unaligned residues shown around alignment, default=30, fasta,ssearch)
-x # Score for match to 'X' (protein) or 'N' (DNA), default -1
-X #,# virtual coordinate of query sequence, library sequences
-z # Statistical estimation strategy
-Z # Effective database size
-1 Sort results by init1 (best single ungapped region)
-3 Search with forward strand only

Running the FASTA programs locally

In addition to running the FASTA programs from the FASTA WWW site fasta.bioch.virginia.edu, you can run the FASTA programs on your workstation from the command line.

For example, 

fasta34 -q gtt1_drome.aa a
runs the exact same search with the drosophila glutathione transferase sequence that you ran in the first exercise.

The FASTA programs can be from the command line, and in interactive mode. A complete list of programs in shown in Table I. In interactive mode, the program prompts for a sequence file name and a library, e.g.

In general, it is easier to do the search from the command line: 

fasta34 -q gtt1_drome.aa q > gtt1_drome.fasta_results
In this case the "-q" indicates that the program should not prompt for any input. After the options, the three command line arguments are 
fasta34 query_file library ktup

FASTA command line options

Search parameters, output formats, and statistical methods can be changed with command line options. With the FASTA programs (as with many Unix programs), command line options MUST preceed any command arguments; thus: 

fasta34 -opt1 -opt2 -opt3 query_file library ktup
A complete set of command line options is shown in Table II. The default parameter values (scoring matrices, gap penalties, etc) are listed in Table III.

The most commonly used program options are "-q" (quiet), and "-S" (ignore low complexity regions). The scoring matrix is set with the "-s" option; gap open and gap extend with "-f/-g".

In general, since FASTA searches may take several minutes (and sometimes much longer), you should always direct the output of the program to a file: 

tfastx34 -q gtt1_drome.aa L > gtt1_tfx_pl.results

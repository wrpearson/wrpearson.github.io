GO - the Gene Ontology

Summary

When I put this exercise together, the amigo.geneontology.org server was down, and this is the server that provides one of the simplest interfaces to the gene ontology.

Instead, we will use www.ebi.ac.uk/QuickGO/ to explore the Gene Ontology for my favorite proteins, the Glutathione transferases. You will use a similar strategy (automated with Python) for your homework.

Overall statistics on GO Go to the QuickGO website, and explore the coverage and evidence codes for the GO annotaions on mouse, fruitfly (Drosophila), and human.

  1. From the QuickGO main page, select the Dataset icon/link, and look at the different resources contributing to GO. The list is a mixture of model organism databases, e.g. MGI (Mouse Genome Informatics), Flybase (Drosophila), RGD (Rat Genome Database), and Yeast (SGD, Sacchromyces Genome Database). Ironically, there is no Human Genome Database.

    In addition, there are annotations provided by non-genome groups, such as Uniprot and InterPro.

  2. Select the Mouse Genome Database (MGI). How many annotations does it provide? Over how many proteins? Does this make sense? How many proteins do you think are in mouse?
  3. Use the filter option to limit the MGI annotations to mouse proteins (taxon). Select Submit after selecting Mouse 10090 taxon.
  4. Select Statistics to look at the distribution evidence codes. What is the most common evidence code? How many annotations are IEA?

    Likewise, use Aspect to see how the annotations and proteins are divided across the three GO ontologies.

    Finally, look at GO Identifier to see what the most common GO-identifiers are for the mouse annotations and mouse proteins. How has the number of proteins annotated in each of the three ontologies changed? Do these numbers agree with the expected number of proteins in mouse?

    What is the most common GO annotation? The most common "specific" GO annotation that is not a cellular location?

  5. Go back to the Data sources page, and select the Uniprot annotations. Again, use the filter option to select Uniprot annotations on Mouse. Now how many protein are annotated? What is the distribution of evidence codes?
  6. Which set of GO-annotations on mouse proteins is more comprehensive? Which do you think is likely to be more reliable? Why?

Looking up a protein

  1. Go back to the main QuickGO main page, and search for P09488. How many proteins are returned? (If you search for GSTM1_HUMAN, you get a very different result).
  2. How many GO annotations are associated with P09488 in each of the three GO ontologies? How many of the annotations are different?
  3. Click on the GO:0004364 annotation supported by direct assay (IDA) and traceable author statement (TAS). What are the "parents" of glutathione transferase activity.
  4. Looking at the list of functions, are there any that are supported only by IEA evidence? Do you think that this annotation is less reliable than the non-IEA assignments? Why or why not?
  5. Use the Download link to download the annotations for this protein. Take a look at the downloaded file.
  6. Look up the P09488 (GSTM1_HUMAN) protein at Amigo. How many GO-annotations do you see? How many are different? Are they the same as QuickGO?

    Which presentation seems more informative?

Homework

The overall goal of the home work is to re-evaluate, using a protein family of your choosing, the results of Nerht et al that argue that paralogs are more similar in function than orthologs.

  1. Starting with a human glutathione transferase, or a human protein from another protein family (e.g. the one you used to do your mid-term project), search the SwissProt database using fasta36 -m 8 or blastp with tabular output to get a list of homologs with E()<1e-6.
  2. Using grep, isolate the _HUMAN protein homologs into one file, and the _MOUSE protein homologs into another.
  3. Write a python script that takes the list of HUMAN or MOUSE paralogs, and extracts the Uniprot Accession (P12345) without a version number, and then uses the URL:

    http://www.ebi.ac.uk/QuickGO/GAnnotation?protein=P09488&format=tsv

    to download a tab-delimited file with GO annotations for each protein For each protein, capture the GO-identifier, and the evidence codes associated with the GO assignment (there can be multiple codes for one GO assignment).

  4. Looking at the human paralogs, make tables that report the number of identical GO-identifiers for the most identical paralog, the second most identical, etc, one for all GO annotations, and a second only for all but IEA, and a third for all annotations (three tables total). At what percent identity do the GO-identifiers begin to differ?
  5. Use your python script to make the same three tables for the set of mouse paralogs.
  6. Finally, compare the human protein to the most similar (possibly orthologous) mouse protein. Again, how many GO identifiers are shared using only experimental evidence, all but IEA, and all annotations.
